# Code Of Conduct

This project follows a practical public-demo conduct policy.

## Expected Behavior

- Keep feedback public-safe and non-sensitive.
- Respect the public/synthetic-only demo boundary.
- Report confusing route decisions, safety gaps, docs gaps, and bugs with enough public context to reproduce.
- Assume route decisions are advice and proof, not permission for external action.

## Unacceptable Behavior

- Posting secrets, credentials, private customer data, company data, exploit payloads against real targets, or regulated data.
- Asking maintainers or agents to execute paid work, public writes, account mutation, supplier dispatch, or private-data processing through the POC.
- Attempting to bypass rate limits, disabled dispatch, input filters, or public-demo boundaries.

## Reporting

Use GitHub Issues only for public, non-sensitive reports. Sensitive reports need a private channel that does not exist yet; stop and wait for that channel before sending sensitive details.
