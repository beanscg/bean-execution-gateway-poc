# Public Launch Packet

Status: not launched. Use this only after verification and live smoke pass.

## Product Claim

Bean Execution Gateway is a gateway to the right agent path. A requester gives an outcome; BEAN routes to an owned agent, a public/open path, or a build decision before anyone pays for compute.

## Current Proof

- Public hosted demo on Render Free.
- Public or synthetic inputs only.
- Zero spend.
- No external writes.
- No external supplier execution.
- Open-demand scan over fixture or read-only public GitHub signals.
- Local packet and static verifier loop.
- Metadata-only feedback loop.

## Hero Metrics

- Executable path rate: share of scanned public signals that become safe local packet candidates.
- Time to ranked paths: wall-clock time from scan to ranked paths.
- Unsafe actions prevented: gates stopped before spend, private data, account creation, public posting, or external submission.

These are proof signals only. They are not revenue, production SLA, or marketplace liquidity claims.

## Demo Script

1. Open the live demo.
2. Click a proof example.
3. Run an open-demand scan.
4. Select a candidate.
5. Build a packet.
6. Run the verifier.
7. Show that external submission remains gated.
8. Record metadata-only feedback.

## Not Ready To Claim

- Live marketplace.
- Paid supplier network.
- Customer traffic readiness.
- Multi-tenant private context.
- Payment settlement.
- Production audit retention.
- Autonomous public posting or external task execution.

## Launch Stop Conditions

- Platform asks for a payment method or paid plan.
- Any private, work, company, customer, secret, credential, local-file, internal, or regulated data is needed.
- Any public post, PR, issue comment, account creation, supplier call, or paid API is required.
- Any claim would imply production readiness, marketplace liquidity, or revenue that does not exist.
