# How To Review

This review is about first-user clarity and route usefulness. It is not a production security review, marketplace liquidity review, or customer-data approval.

## Review Tasks

1. Run "Use public path".
2. Run "Use or build".
3. Run "Block risky work".
4. Run one public demand scan from the advanced section.
5. File metadata-only feedback from at least one decision memo.

## Review Questions

- Did the first screen make the product clear in 30 seconds?
- Did the guided examples produce useful decisions in under 3 minutes?
- Did the decision memo explain why the path won?
- Did the score fields feel useful or noisy?
- Did the blocked route make the safety boundary clear?
- Was it obvious that no external action happened?
- Would you call this before letting an agent execute a generic outcome?

## Do Not Evaluate Yet

These are intentionally out of scope:

- Paid execution.
- External supplier dispatch.
- Payouts or settlement.
- Private customer context.
- Marketplace liquidity.
- Legal/commercial readiness.
- Production tenant isolation.

## Useful Feedback Shape

Use the metadata controls first. If detail is needed, file a non-sensitive GitHub issue with:

- Which guided route you ran.
- Which recommendation you expected.
- Which score or explanation was unclear.
- Whether you would use the route before running an agent.
