# Public Beta Positioning

## One-Liner

BEAN is a gateway to the right agent path: give it an outcome, and it decides whether to use an owned agent, use a public/open path, or build a new agent before anyone pays for compute.

## Category

Execution gateway. Not a marketplace yet. Not an agent runtime. Not a model wrapper. Not a workflow automation platform.

## First User

The first user is a technical operator who already asks agents for broad outcomes and wants a safer answer to: "what is the best path to get this done?"

Good early users:

- Agent builders.
- Devtool users.
- Automation-heavy founders/operators.
- Engineers who maintain public projects.
- People who understand public/synthetic demo boundaries.

Bad early users:

- Users with private customer data.
- Users needing guaranteed execution.
- Users looking for paid supplier fulfillment.
- Users who need regulated, financial, legal, medical, trading, or security-exploitation workflows.

## Primary Promise

BEAN turns an outcome request into a ranked path with proof and stop conditions.

It can say:

- Use this owned/local agent path.
- Use this public/open path.
- Build a new agent because no available path is strong enough.
- Stop because the request needs money, private data, account mutation, public posting, or external supplier execution.

## What The Demo Proves

- Route decisions.
- Public-demand scanning.
- Local proof packets.
- Static verifier reports.
- Metadata-only feedback.
- Safety gates before spend or external execution.
- V1 and V2 product contract maps.

## What The Demo Does Not Prove

- Live marketplace liquidity.
- Paid supplier quality.
- Customer/private data readiness.
- Production durability.
- Payment settlement.
- Revenue.
- SLA or uptime.

## FAQ: Why Not Just Use ChatGPT, Cursor, Or Claude?

Those tools can do work. BEAN decides the path before work starts. It is useful when the prompt is broad, the agent needs to decide whether to use an existing agent or build one, and the operator wants proof that unsafe or expensive steps were gated.

BEAN should complement models and agent tools, not replace them.

## FAQ: What Happens To My Data?

The public demo is for public or synthetic inputs only. Hosted mode does not intentionally log or persist request bodies in app code. Feedback endpoints accept enumerated metadata and reject free-text fields. Public GitHub issues are public, so issue feedback must not include sensitive data.

## Safe Claims

- BEAN is a public proof of concept.
- BEAN is an execution gateway.
- BEAN routes outcomes to owned-agent, public-path, or build-decision paths.
- BEAN blocks spend, private data, public posting, account mutation, and external supplier execution in the current demo.
- BEAN exposes public-learning and product-contract endpoints.

## Forbidden Claims

- Do not claim BEAN is a live marketplace.
- Do not claim BEAN has paid suppliers.
- Do not claim BEAN can process customer/private data.
- Do not claim BEAN can execute work externally.
- Do not claim BEAN is production-ready.
- Do not claim revenue, liquidity, or SLA metrics.
